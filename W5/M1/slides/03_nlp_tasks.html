Natural Language Processing (NLP)

NLP bridges the gap between computers and human language. It combines elements of:

  1. computer science,
  2. linguistics,
  3. artificial intelligence

Recent success in NLP has been driven by Deep Learning approach which is based on massive datasets and models that can learn from data.

NLU & NLG

Natural Language Understanding (NLU): extract meaning, intention, emotion, importance, and correlation between words/texts/speech.

  • Examples: Classification, NER, and ASR

Natural Language Generation (NLG): generate the most likely sequence given previous sequence.

  • Examples: Question Answering, Summarization, Translation, and Coding.

NLU Tasks

Foundational discriminative tasks in natural language understanding (NLU):

Task Modality Output Models Datasets
text-classification Text Label and confidence score 🔗 🔗
token-classification Text Tokens with entity tags (e.g., PER, LOC) 🔗 🔗
question-answering Text Extracted answer span from a context 🔗 🔗
zero-shot-classification Text Labels matched without task-specific training 🔗 🔗

NLG Tasks

Task Modality Output Models Datasets
summarization Text Condensed version of the input text 🔗 🔗
translation Text Translated text in the target language 🔗 🔗
text-generation Text AI-generated continuation 🔗 🔗
text-to-speech Audio Convert text to spoken audio 🔗 🔗
automatic-speech-recognition Audio Transcribed text from audio 🔗 🔗
image-to-text Image Generate text descriptions of images 🔗 🔗
image-text-to-text Multimodal Respond to an image based on a text prompt 🔗 🔗

1. Text Classification

Text Classification is the process of assigning a category or label to a piece of text.

Applications

Spam Detection (Spam / Not Spam) in emails and messages:

“Congratulations! You’ve won a $1000 Walmart gift card. Click here to claim your prize.”

Content Moderation essential part of any online community.

  • Fake reviews (تقييمات مزيفة)
  • Fake news (أخبار مزيفة)
  • Fake accounts (حسابات مزيفة)
  • Inappropriate content (محتوى غير لائق)

1.1 Sentiment Analysis

Sentiment Analysis identifies the emotional tone of text.

  • Positive: “This coffee is amazing!”
  • Negative: “My order arrived cold and late.”
  • Neutral: “The store opens at 8 AM.”

If we don’t specify a model, a default model will be used:

from transformers import pipeline

classifier = pipeline(task="sentiment-analysis")
output = classifier([
    "I've been waiting for a HuggingFace course my whole life.",
    "I hate this so much!"
])

print(output)
[{'label': 'POSITIVE', 'score': 0.9598046541213989}, {'label': 'NEGATIVE', 'score': 0.9994558691978455}]

if output[0]['label'] == 'POSITIVE':
    print("the user likes the product")
the user likes the product

1.3 Zero-shot Classification

Models trained with NLI can do zero-shot classification; i.e., rather than being specifically trained on a fixed set of classes, they can take candidate_labels at inference time, and be able to output class probability on them:

from transformers import pipeline

zero_shot_classifier = pipeline("zero-shot-classification")
output = zero_shot_classifier(
    "This is a course about the Transformers library",
    candidate_labels=["education", "politics", "business"],
)

print(output)
{
    'sequence': 'This is a course about the Transformers library',
    'labels': ['education', 'business', 'politics'],
    'scores': [0.844595193862915, 0.11197695881128311, 0.04342786595225334]
}

output = zero_shot_classifier(
    "I need my money back",
    candidate_labels=[
        "Refund",
        "Question",
        "Complain"
    ],
)

label = output['labels'][0]

if label == 'Refund':
    print("Trigger workflow for getting a refund")
elif label == 'Complain':
    print("Escalate to customer support manager")
elif label == 'Question':
    print("Direct user to the FAQ documentation")
else:
    print(f"Categorized as: {label}")
{
    'sequence': 'I need my money back',
    'labels': ['Refund', 'Complain', 'Question'],
    'scores': [0.5873205661773682, 0.23744213581085205, 0.17523722350597382]
}

3. Named-entity Recognition (NER)

Named entity recognition (NER) aims to extract entities in a piece of text into predefined categories such as: personal names, organizations, locations, and quantities.

Task name: token-classification.

Example: GLiNER.

Applications

  • Building knowledge graphs for Arabic content.
  • Enhancing search and recommendation systems with entity-aware features.
from transformers import pipeline

ner = pipeline(
    task="token-classification",
    aggregation_strategy="simple"
)
ner("My name is Sylvain and I work at Hugging Face in Brooklyn.")

7. Text Generation (LLM)

LLM .generate() method auto-regressively generates the next sequence given some initial prompt. Generation can be controlled:

from transformers import pipeline, GenerationConfig

generation_config = GenerationConfig(
    max_new_tokens=50,
    temperature=1.5
)

generator = pipeline(
    task="text-generation",
    model="HuggingFaceTB/SmolLM2-360M",
    generation_config=generation_config,
)

Common Generation Options

See: Common Options:

  • max_new_tokens (int): Controls the maximum number of tokens generated. Make sure to set it explicitly—defaults are usually small.
  • num_beams (int): Activates beam search when set above 1. Useful for input-grounded tasks. See this guide.
  • repetition_penalty (float): Set to greater than 1.0 to discourage the model from repeating itself.
  • temperature (float): Influences randomness of generation. High values (>0.8) make outputs more creative; low values (<0.4) make them more focused and predictable.
output = generator(
    "In this course, we will teach you how to",
    max_new_tokens=30,
    num_return_sequences=3,
)
print(output)

LLM for Specific Tasks

from transformers import pipeline

checkpoint = "HuggingFaceTB/SmolLM2-360M"
generator = pipeline(
    "text-generation",
    model=checkpoint,
    device="auto"
)


def emotion_classifier(batch: list[str], emotions: list[str]) -> list[str]:
    """
    Classifies the emotion of each text.
    Accepts a list of strings (batch).
    """
    emotion_options = ", ".join(emotions)
    prompts = [
        (
            f"Select the primary emotion expressed in the following text from the provided list. Respond with only one emotion.\n\n"
            f"Possible emotions: {emotion_options}\n"
            f"Text:\n{t}\n\nEmotion:"
        )
        for t in batch
    ]
    results = generator(prompts, max_new_tokens=8, do_sample=False)
    outputs = [r["generated_text"] for r in results]
    return outputs


def question_answerer(batch: list[str]) -> list[str]:
    """
    Answers questions about the provided texts using text generation.
    Accepts a list of strings (batch).
    """
    prompts = [
        (
            "Given the following text, provide a concise answer to the main question or information presented.\n\n"
            f"Text:\n{t}\n\nAnswer:"
        )
        for t in batch
    ]
